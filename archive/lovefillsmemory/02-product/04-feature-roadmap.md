# Feature Roadmap｜功能路線圖

Version 1.0

---

# Purpose｜目的

🇺🇸

This document defines what we build now, next, and never.

It protects focus.

It ensures we build for caregivers, not for feature complexity.

---

🇹🇼

本文件定義：

現在做什麼、下一步做什麼、以及永遠不做什麼。

它的目的是保護產品專注度，

確保我們是為照顧者而設計，而不是為功能複雜度而設計。

---

🇺🇸

### 1. Daily Care Entry
- Text input
- Voice input
- Photo input

---

### 2. AI Summary
- Extract key care moments
- Summarize daily activity
- Store structured record

---

### 3. Emotional Response
- AI responds with supportive message
- Non-judgmental tone
- Reduces emotional burden

---

### 4. Simple Timeline View
- List of daily entries
- Chronological order
- No complex filters

---

🇹🇼

### 1. 每日照護記錄

- 語音輸入

### 3. 情緒回應

# ❌ Never Build (Very Important)

🇺🇸

We explicitly DO NOT build:

- Social media features
- Likes / comments / engagement systems
- Complex dashboards
- Gamification systems
- Advertising systems
- Medical diagnosis tools

---

🇹🇼

我們明確不做：

- 社群功能
- 按讚 / 留言 / 互動
- 複雜儀表板
- 遊戲化設計
- 廣告系統
- 醫療診斷工具

---

# 🧭 Product Rule

🇺🇸

If a feature does not reduce caregiver burden or increase clarity:

We do not build it.

---

🇹🇼

如果一個功能，

不能減輕照顧者負擔或提升清晰度，

我們就不做。

---

# 🎯 MVP Success Definition

🇺🇸

We succeed if a caregiver says:

> “This helped me feel less alone today.”

---

🇹🇼

如果照顧者說：

> 「今天，我沒有那麼孤單了。」

我們就成功了。
